  • About The Website

    Hello, welcome to my web site!
    My name is Alex. I enjoy programming in Java. I love learning new things about Java and sharing my experience with everyone. Most of applications and websites are dependent on Java. Right from game consoles to cell phones java is ruling the technical market. My website aims to provide detailed articles and tutorials about design patterns in Java which are written and developed from a developer perspective. Most of the articles include examples which are shown in this website and are available in my repository.
    Everyone can download for free from https://github.com/malexj and analyze them. Ask all the questions you need to. Don't be afraid to ask for help.

    Why do I do all this?

    Writing articles for this website provides me a chance to investigate and get my hands dirty with technologies that I never used before or I just heard of. I do lots of reading and experiments before I write an article and this takes time. But so far it is worth the effort.

    List of technologies used in the creation of the website referenced below...

  • Web components used when creating the site:

    AngularJS
    AngularJS is an open-source web application framework mainly maintained by Google and by a community of individuals and corporations to address many of the challenges encountered in developing single-page applications.

    Apache Commons
    The Apache Commons is a project of the Apache Software Foundation, formerly under the Jakarta Project. The purpose of the Commons is to provide reusable, open source Java software. The Commons is composed of three parts: proper, sandbox, and dormant.
    The usage list is the following:
    commons-io
    commons-fileupload
    commons-codec
    commons-dbcp2
    maven

    Thymeleaf
    Thymeleaf is a Java XML/XHTML/HTML5 template engine that can work both in web (Servlet-based) and non-web environments.

    OpenShift
    OpenShift is Red Hat's Platform-as-a-Service (PaaS) that allows developers to quickly develop, host, and scale applications in a cloud environment. With OpenShift you have a choice of offerings, including online, on-premise, and open source project options.

    JBoss
    The JBoss Enterprise Application Platform (or JBoss EAP) is a subscription-based/open-source Java EE-based application server runtime platform used for building, deploying, and hosting highly-transactional Java applications and services.

  • Spring
    The Spring Framework is an application framework and inversion of control container for the Java platform. The framework's core features can be used by any Java application, but there are extensions for building web applications on top of the Java EE platform.

    Used the following component:
    Spring Core
    Spring MVC
    Spring Security
    Spring Data-Jpa
    Spring Transaction
    Spring-Test

  • Hibernate
    Hibernate ORM (Hibernate in short) is an object-relational mapping framework for the Java language. It provides a framework for mapping an object-oriented domain model to a relational database.

    PostgreSQL
    PostgreSQL is an object-relational database management system (ORDBMS) with an emphasis on extensibility and standards-compliance.

    Dozer
    Dozer is a Java Bean to Java Bean mapper that recursively copies data from one object to another.

  • JUnit
    JUnit is a unit testing framework for the Java programming language. JUnit has been important in the development of test-driven development.

    JWebUnit
    JWebUnit is a Java-based testing framework for web applications. It wraps existing testing frameworks such as HtmlUnit and Selenium with a unified, simple testing interface to allow you to quickly test the correctness of your web applications.

    Mockito
    Tasty mocking framework for unit tests in Java.

    DbUnit
    DbUnit is a JUnit extension targeted at database-driven projects that, among other things, puts your database into a known state between test runs.

    JaCoCo
    JaCoCo is a free code coverage library for Java, which has been created by the EclEmma team based on the lessons learned from using and integration existing libraries for many years.
